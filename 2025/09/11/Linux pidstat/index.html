Linux pidstat

发表于2025-09-11|更新于2025-09-11|Linux性能
|浏览量:

pidstat 是 Linux 下 sysstat 工具集中的一个命令，用来监控指定进程（PID）的各种运行指标，常用于排查性能问题。它比 top 更细粒度，可以按进程或线程输出统计信息。

一、基本语法

  • 时间间隔：两次采样之间的秒数
  • 次数：采样的次数
  • 如果不加时间间隔和次数，只输出一次快照

二、常用选项

  • **-u**：显示 CPU 使用情况（默认）
  • **-r**：显示内存使用情况（RSS、VSZ、内存比例等）
  • **-d**：显示 I/O 使用情况（读写速率）
  • **-w**：显示上下文切换情况（自愿/非自愿）
  • **-t**：显示线程级别统计（默认是进程级别）
  • **-p <pid>**：指定进程 PID（否则显示所有进程）
  • **-h**：以更易读的方式显示时间（hh:mm:ss 格式）

三、示例

  1. 监控所有进程的 CPU 使用情况，每隔 2 秒，采集 5 次
  1. 监控某个进程（PID=1234）的 CPU 和内存情况
  1. 监控所有进程的 I/O 使用情况
  1. 查看线程级别的 CPU 使用情况
  1. 统计上下文切换

四、输出字段解释（示例：pidstat -u）

  • UID：进程所属用户 ID
  • PID：进程号
  • %usr：用户态 CPU 使用率
  • %system：内核态 CPU 使用率
  • %guest：运行虚拟 CPU 的时间
  • %CPU：进程总 CPU 使用率
  • CPU：进程运行在哪个 CPU 上
  • Command：命令名称

五、常见应用场景

  • 精确定位某个进程的 CPU 消耗（比 top 更直观）
  • 分析进程的 内存占用趋势
  • 查看进程的 磁盘 I/O 行为
  • 调查进程的 上下文切换频率（排查线程切换过多问题）
  • 按 线程 维度分析性能瓶颈

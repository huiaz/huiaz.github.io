rune 和 byte

核心区别：代表的“单位”不同

简洁地说：

  • byte 代表一个字节 (8 bits)。
  • rune 代表一个 Unicode 码点（即一个字符）。

详细解析

1. byte 类型

  • 底层类型： byte 是 uint8 的别名。

    这意味着 byte 类型就是一个无符号的 8 位整型，其值范围是 0 到 255。

  • 代表意义：

    • 一个字节： 它是计算机内存中可寻址的最小单位。
    • 原始数据： byte 通常用于处理原始的二进制数据流，比如网络传输中的数据包、文件内容、图片数据等。
    • ASCII 字符： 对于 ASCII 字符集（英文字母、数字、常见符号），一个字符恰好占用一个字节，所以 byte 也可以表示这些单一字节的字符。

  • Go 字符串与 byte：

    • 在 Go 语言中，**字符串的底层是 UTF-8 编码的字节序列([]byte)**。len(someString) 返回的是字符串的字节数，而不是字符数。
    • 当你直接对字符串进行索引访问时（如 s[0]），你得到的是一个 byte 值，代表该索引位置的原始字节。
    • 当使用 for i, v := range someString {} 循环字符串时，如果 v 被推断为 byte，那么循环会按字节进行，这对于多字节字符（如汉字）是不正确的。然而，Go 语言的 for range 对字符串有特殊处理，它会按 rune 来迭代。

2. rune 类型

  • 底层类型： rune 是 int32 的别名。

    这意味着 rune 类型是一个 32 位整型，其值范围足以覆盖所有可能的 Unicode 码点（从 U+0000 到 U+10FFFF）。

  • 代表意义：

    • 一个 Unicode 码点： 它是对单个字符的抽象，无论这个字符在 UTF-8 编码中占用多少个字节（1到4个字节）。
    • 字符处理： rune 主要用于处理人类可读的文本，特别是当涉及到多语言（如中文、日文、韩文）或特殊符号（如表情符号）时。

  • Go 字符串与 rune：

    • 当你想要获取字符串的“字符”长度时，你需要将字符串转换为 []rune 切片，再获取其长度：len([]rune(someString))。
    • for range 循环字符串时，它会自动解码 UTF-8 字节序列，每次迭代返回一个 rune（字符）和一个字节索引（该字符在原字符串中的起始字节位置）。这是 Go 语言处理 Unicode 字符串的惯用且推荐的方式。

对比总结

特性/方面 byte (uint8) rune (int32)
含义 一个 8 位的字节值 一个 Unicode 码点（即一个字符）
大小 1 字节 4 字节
范围 0 到 255 0x0 到 0x10FFFF (所有 Unicode 码点范围)
主要用途 原始二进制数据、网络传输、文件 I/O、ASCII 字符 多语言文本处理、计算字符数、按字符遍历、国际化应用
与字符串关系 Go 字符串底层是 []byte（UTF-8 编码），len(s) 返回字节数，s[i] 返回字节 for range 字符串默认迭代 rune，len([]rune(s)) 返回字符数
典型场景 io.Reader/Writer, net.Conn, encoding/binary unicode 包函数 (如 unicode.IsLetter), strings 包中的字符操作函数

理解 rune 和 byte 的区别，尤其是 Go 字符串的 UTF-8 编码特性，是编写正确且健壮的 Go 文本处理代码的关键。在处理任何可能包含非 ASCII 字符的文本时，都应优先考虑使用 rune 来进行字符级别的操作。

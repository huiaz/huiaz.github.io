推荐使用方案

最推荐使用 map[T]struct{} 方案，原因如下：

  1. 内存效率高：struct{} 不占用内存空间
  2. 性能优秀：map 的查找、插入、删除操作平均时间复杂度为 O(1)
  3. 类型安全：使用泛型确保类型安全
  4. 标准库兼容：不依赖第三方库
  5. 语义清晰：明确表达了 set 的概念

这种实现方式在实际的后端开发中非常常见，特别适用于去重、成员检查、缓存管理等场景。
